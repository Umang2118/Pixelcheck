## 🚨 The Problem: Fraud in Food Delivery

Food delivery giants like Swiggy and Zomato process millions of orders daily, ensuring customer satisfaction with robust refund and replacement policies. However, a new wave of fraud is exploiting these policies. Malicious actors are utilizing advanced Generative AI to create hyper-realistic fake images of food—depicting stale meals, foreign objects (like insects), or incorrect orders. These AI-generated images are submitted as "proof" to falsely claim refunds. 

**The Result:** Food delivery platforms face massive, unjustified financial losses.

**The Solution:** PixelCheck provides an immediate, automated defense. By integrating our REST API into the customer support workflow, platforms can instantly scan user-uploaded claim images. If the image is flagged as AI-generated, the fraudulent claim is automatically flagged or rejected, saving the company significant revenue while protecting the integrity of their platform.

## 📖 Overview
PixelCheck is a full-stack platform designed to identify whether an image is authentic (real) or generated by AI (fake). Built with **PyTorch** for model inference and **Flask** for the backend, it features a sleek user interface for image uploads, historical tracking of predictions, user authentication, and a dedicated REST API endpoint for integration into external systems.

## ✨ Features
- **Accurate AI Detection**: Leverages a fine-tuned deep learning model to predict whether an image is Real or AI-Generated.
- **Web UI**: A clean, responsive interface where users can drag & drop images for instant analysis.
- **RESTful API**: Easily integrate the detection engine into backend systems using the `/api/predict` endpoint.
- **User Authentication**: Secure sign-up and login functionality powered by Flask-Login and bcrypt.
- **Analysis History**: Logged-in users can view a history of their past image scans and confidence scores.
- **Extensible Architecture**: Code is split neatly between model definition, training logic (`src/`), and web application logic (`app.py`).

## 🛠️ Technology Stack
- **Backend**: Python, Flask, SQLAlchemy, SQLite
- **Machine Learning**: PyTorch, Torchvision, Pillow
- **Frontend**: HTML5, CSS3, Jinja2 Templates
- **Security**: Flask-Bcrypt, Werkzeug

## 🚀 Getting Started

### Prerequisites
- Python 3.8+
- [Git](https://git-scm.com/)

### Installation
1. **Clone the repository:**
   ```bash
   git clone https://github.com/Umang2118/Fake-Ai-Image-Detection.git
   cd Fake-Ai-Image-Detection
   ```

2. **Create and activate a virtual environment (optional but recommended):**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install the dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Prepare the Model:**
   - Ensure your trained model weights (`best_model.pth`) are placed inside a `models/` directory in the project root. The app automatically detects weights in this folder.

5. **Run the Application:**
   ```bash
   python app.py
   ```
   The app will start running at `http://127.0.0.1:5000/`.

## 🔌 API Reference

### Predict Image Authenticity
**Endpoint:** `POST /api/predict`

Integrate PixelCheck directly into your applications.

**Request:**
- Content-Type: `multipart/form-data`
- Body: `file` (The image file you want to analyze)

**Response (JSON):**
```json
{
  "class": "AI Generated",
  "confidence": 98.45,
  "confidence_display": "98.45%",
  "filename": "sample.jpg",
  "is_fraud": true,
  "recommendation": "REJECT_CLAIM"
}
```

## 🧠 Model Training
If you wish to train the model from scratch on your own dataset:
1. Prepare your dataset format.
2. Configure parameters in `src/train.py`.
3. Run the training script:
   ```bash
   python src/train.py
   ```
