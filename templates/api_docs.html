🎯 Overview

The PixelCheck API allows delivery platforms to automatically verify whether a customer-submitted food claim image is genuine or AI-generated. When a customer opens a refund claim, your backend calls our API with the uploaded image. We return a class, confidence score, and a recommendation (APPROVE_CLAIM or REJECT_CLAIM) so your system can act immediately.

All images are processed in memory and are never stored on our servers. Average response time is under 2 seconds.

📡 Endpoint

POST /api/predict

Accepts a multipart form file upload and returns a JSON classification result.

📤 Request Format

Content-Type: multipart/form-data

Field Type Required Description
file File ✓ Yes The image to classify. Supports JPG, JPEG, PNG. Max 10 MB.

📥 Response Format

200 OK

{
  "class": "AI Generated",
  "is_fraud": true,
  "confidence": 97.43,
  "confidence_display": "97.43%",
  "filename": "claim_photo.jpg",
  "recommendation": "REJECT_CLAIM"
}
Field Type Description
class string "AI Generated" or "Real"
is_fraud boolean true if the image is detected as AI-generated
confidence float Model confidence from 0.00 to 100.00
confidence_display string Human-readable confidence string
filename string Sanitized filename of the uploaded image
recommendation string REJECT_CLAIM or APPROVE_CLAIM

⚠️ Error Codes

Code Meaning Body
400 No file uploaded or empty filename {"error": "No file part or empty filename"}
503 Model not loaded on server {"error": "Model not loaded — weights missing on server"}
500 Internal processing error {"error": "<exception message>"}

Python (requests)

import requests

with open("claim_photo.jpg", "rb") as f:
    response = requests.post(
        "https://www.pixel-check.com/api/predict",
        files={"file": ("claim_photo.jpg", f, "image/jpeg")}
    )

result = response.json()
if result["is_fraud"]:
    print(f"Claim REJECTED: {result['class']} ({result['confidence_display']})")
else:
    print(f"Claim APPROVED: Image appears real ({result['confidence_display']})")

JavaScript (fetch)

const formData = new FormData();
formData.append('file', imageFile);

const res = await fetch('https://www.pixel-check.com/api/predict', {
  method: 'POST',
  body: formData
});
const result = await res.json();
console.log(result.recommendation); // "REJECT_CLAIM" or "APPROVE_CLAIM"

Java (OkHttp)

import okhttp3.*;
import java.io.File;

public class PixelCheckClient {
    public static void main(String[] args) throws Exception {
        OkHttpClient client = new OkHttpClient().newBuilder().build();
        
        RequestBody body = new MultipartBody.Builder().setType(MultipartBody.FORM)
                .addFormDataPart("file", "claim_photo.jpg",
                        RequestBody.create(MediaType.parse("image/jpeg"), 
                        new File("/path/to/claim_photo.jpg")))
                .build();
                
        Request request = new Request.Builder()
                .url("https://www.pixel-check.com/api/predict")
                .method("POST", body)
                .build();
                
        Response response = client.newCall(request).execute();
        System.out.println(response.body().string());
    }
}

🔄 Integration Workflow for Swiggy / Zomato

Here is the recommended end-to-end claim verification workflow for delivery platforms:

  1. Customer opens a refund claim and uploads one or more food images.
  2. Your backend receives the image (before it reaches the human support agent).
  3. Your backend calls POST /api/predict with the image.
  4. PixelCheck responds in <2 seconds with recommendation.
  5. If REJECT_CLAIM: auto-flag or auto-deny the claim with a user-facing explanation.
  6. If APPROVE_CLAIM and confidence > 90%: pass to a support agent as pre-verified genuine.
  7. If confidence < 90%: always escalate to a human agent for review.
